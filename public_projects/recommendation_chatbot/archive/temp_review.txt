# 학습 데이터 처리 및 세션 관리

## 1. 학습 데이터 수집 및 처리 플로우

### 데이터 수집 파이프라인
- 실시간: 사용자 입력 → NLU 처리 → DB 저장 → 검증 큐
- 일일 배치: 데이터 정제 → 학습 데이터셋 생성 → 모델 재학습 → A/B 테스트

### 학습 데이터 테이블
- nlu_training_data: 원본 텍스트, 예측 결과, 정답 라벨, 학습 상태
- recommendation_training_data: 사용자 특징, 추천 결과, 사용자 반응

## 2. 세션 관리 및 데이터 영속성

### 세션 라이프사이클
- 시작: 세션 ID 생성, Redis 초기화
- 진행 중: 대화는 Redis(24시간), 중요 이벤트는 즉시 DB
- 종료: 30분 무활동 또는 창 닫기
- 종료 시: 요약 생성 → DB 저장 → 학습 데이터 추출 → Redis 정리

### 데이터 영속성
- session_summaries: 세션 요약 정보
- conversation_snapshots: 중요 대화만 영구 보관

## 3. 데이터 보관 정책
- 실시간 대화: 24시간 (Redis)
- 세션 요약: 1년
- 학습 데이터: 2년
- 사용자 프로필: 탈퇴 후 30일
